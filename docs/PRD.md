# 小說閱讀器系統 PRD (產品需求文件)

## 1. 專案概述
- **背景與動機**：市面上的小說散落各處，讀者難以在單一平台上找到所有想看的小說，也難以統一管理自己的閱讀進度。本系統期望將所有已上市的小說蒐集在一個系統中，提供讀者一個方便、統一的閱讀體驗。
- **目標用戶**：喜愛閱讀小說的讀者。
- **核心價值主張**：提供最齊全的小說書庫，以及便利的閱讀與追蹤紀錄功能，讓讀者享受無縫的閱讀體驗。

## 2. 功能需求
以下為系統主要功能與對應的使用者故事：

1. **閱讀小說**
   - 作為 讀者，我希望 能夠在系統上直接閱讀小說內容，以便 我可以隨時隨地享受閱讀樂趣。
2. **閱讀紀錄與進度追蹤**
   - 作為 讀者，我希望 系統能自動紀錄我已經閱讀過的小說及當前進度（如章節），以便 我下次能快速接續閱讀。
3. **搜尋與篩選小說**
   - 作為 讀者，我希望 能夠透過書名、作者關鍵字或分類來搜尋小說，以便 我能快速找到想看的特定書籍。
4. **收藏喜愛的小說 (書單)**
   - 作為 讀者，我希望 能夠將喜歡的小說加入我的收藏清單，以便 我未來可以方便地尋找或重溫。
5. **熱門排行榜與分類瀏覽**
   - 作為 讀者，我希望 能看到當前熱門的小說排行榜與各種分類，以便 我在不知道看什麼時能有參考依據。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask
  - 前端模板：Jinja2
  - 資料庫：SQLite
- **效能與安全考量**：
  - 系統需能承受基本的並發請求，小說文章內容需快速載入（例如支援分頁或分章節載入）。
  - 密碼需經過安全雜湊（Hash）處理後儲存。
  - 需要防範常見網頁漏洞，如 SQL Injection 與 XSS。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**
  - 使用者註冊、登入與登出系統
  - 小說清單展示與基本內文閱讀頁面
  - 自動或手動紀錄閱讀進度
- **Should Have (應該有)**
  - 關鍵字搜尋與分類篩選功能
  - 加入收藏夾（書單）功能
- **Nice to Have (可以有)**
  - 排行榜系統
  - 評論與評分系統
  - 閱讀模式自訂（夜間模式、字體大小調整）
